Maintenance et mise à jour d'un serveur Linux

Introduction

Ce guide vous fournira les outils et commandes nécessaires pour effectuer une maintenance efficace de votre serveur Linux, en assurant sa sécurité, sa performance et sa fiabilité.

Outils de maintenance courants

  • htop : Surveillance des processus en temps réel.
  • df : Affiche l'espace disque disponible.
  • du : Analyse l'utilisation de l'espace disque par dossier.
  • journalctl : Consultez les journaux système.
  • top : Affiche les tâches en cours et la charge système.

Commandes pour les mises à jour

  1. Pour les systèmes basés sur Debian (Ubuntu, etc.) : sudo apt update && sudo apt upgrade

    Mettez à jour la liste des paquets et appliquez les mises à jour disponibles.

  2. Pour les systèmes basés sur Red Hat (CentOS, Fedora, etc.) : sudo yum update ou sudo dnf update

    Mettez à jour les paquets et le système.

  3. Mettre à jour le noyau Linux : sudo apt install linux-generic (Debian) ou sudo yum install kernel (Red Hat).

Gestion des services

Utilisez systemctl pour gérer les services :

  • sudo systemctl start [service] : Démarrer un service.
  • sudo systemctl stop [service] : Arrêter un service.
  • sudo systemctl restart [service] : Redémarrer un service.
  • sudo systemctl status [service] : Vérifier l'état d'un service.
  • sudo systemctl enable [service] : Activer un service au démarrage.

Vérification et sauvegardes

  • Surveiller les journaux : Utilisez journalctl -xe pour diagnostiquer les erreurs.
  • Créer des sauvegardes : Configurez des sauvegardes régulières avec rsync ou des outils comme timeshift.
  • Testez les sauvegardes : Assurez-vous que vos sauvegardes sont utilisables en effectuant des restaurations périodiques.
