Tutoriel : Mise en œuvre du Spanning Tree Protocol (STP)

Introduction

Le Spanning Tree Protocol (STP) est un protocole réseau essentiel conçu pour prévenir les boucles dans un réseau Ethernet commuté. Les boucles peuvent provoquer une surcharge des switches et des interruptions de service. STP permet à un réseau de rester redondant tout en désactivant dynamiquement les chemins redondants inutilisés.

Dans ce tutoriel, nous allons expliquer les concepts de base de STP, ainsi que fournir des étapes pour configurer ce protocole sur des switches.

Concepts de base

  • STP (Spanning Tree Protocol) : STP crée une topologie logique sans boucle en désactivant certains liens redondants tout en maintenant la redondance pour la tolérance aux pannes.
  • Bridge ID : Un identifiant unique pour chaque switch, composé d'une priorité (valeur par défaut : 32768) et de l'adresse MAC du switch.
  • Root Bridge : Le switch élu comme point central de la topologie STP. Tous les chemins vers le Root Bridge sont calculés avec le coût le plus bas.
  • Port States : STP utilise plusieurs états de port :
    • Blocking : Aucun trafic de données n'est transmis.
    • Listening : Le switch écoute les BPDU (Bridge Protocol Data Units).
    • Learning : Le switch apprend les adresses MAC.
    • Forwarding : Le trafic de données est transmis.
    • Disabled : Le port est administrativement désactivé.

Prérequis

  • Switches compatibles avec STP (Cisco, HP, etc.).
  • Connaissances de base sur les VLANs et les topologies réseaux.

Étapes pour configurer STP

Exemple avec des switches Cisco (IOS)

  1. Vérifier l'état actuel de STP : 
    Switch> enable
Switch# show spanning-tree
  2. Configurer la priorité du Root Bridge : Pour élire un switch comme Root Bridge, configurez une priorité inférieure (par exemple, 4096) : 
    Switch# configure terminal
Switch(config)# spanning-tree vlan 1 priority 4096
  3. Configurer les ports STP :
    • Définir un port en mode Edge (PortFast) : Ce mode est utilisé pour les ports connectés à des hôtes finaux, ce qui permet de réduire le temps de convergence : 
      Switch(config)# interface FastEthernet0/1
Switch(config-if)# spanning-tree portfast
    • Activer BPDU Guard : Pour protéger les ports Edge contre les connexions non autorisées de switches : 
      Switch(config-if)# spanning-tree bpduguard enable
  4. Configurer la redondance et les chemins de backup : Si vous utilisez plusieurs liens redondants, assurez-vous que STP bloque les chemins supplémentaires pour éviter les boucles.
  5. Vérifier la configuration : 
    Switch# show spanning-tree detail

Commandes supplémentaires utiles

  • Modifier le coût d'un port : Pour influencer le choix du chemin STP : 
    Switch(config)# interface FastEthernet0/2
Switch(config-if)# spanning-tree cost 19
  • Modifier la priorité d'un port : Pour préférer un port à un autre : 
    Switch(config)# interface FastEthernet0/2
Switch(config-if)# spanning-tree port-priority 64
  • Activer Rapid Spanning Tree Protocol (RSTP) : Pour une convergence plus rapide : 
    Switch(config)# spanning-tree mode rapid-pvst

Conclusion

Le Spanning Tree Protocol est indispensable pour garantir un réseau commuté sans boucle tout en maintenant une redondance efficace. En suivant les étapes ci-dessus, vous pouvez configurer STP pour assurer une topologie réseau stable et performante. Adaptez les commandes à vos besoins spécifiques et consultez la documentation de votre équipement pour des configurations avancées.
