Tutoriel : Configuration de liens trunk pour le transport des VLANs entre switches

Introduction

Le trunking est une technique utilisée pour permettre le transport des informations de plusieurs VLANs sur une même liaison entre deux switches. Cela est essentiel dans un réseau où plusieurs VLANs coexistent, car cela garantit que les communications entre les VLANs restent isolées tout en transitant efficacement par un lien commun.

Dans ce tutoriel, nous allons expliquer les concepts de base du trunking et fournir des étapes pratiques pour configurer des liens trunk sur des switches.

Concepts de base

  • VLAN (Virtual Local Area Network) : Un VLAN permet de segmenter un réseau physique en plusieurs réseaux logiques indépendants.
  • Trunk : Un lien trunk est une connexion entre deux switches (ou entre un switch et un routeur) qui transporte des trames provenant de plusieurs VLANs.
  • 802.1Q : C'est le standard utilisé pour insérer une balise VLAN dans une trame Ethernet. Cette balise identifie le VLAN d'origine de chaque trame.

Prérequis

  • Deux switches configurables (Cisco, HP, Juniper, etc.).
  • Les VLANs doivent déjà être créés sur les switches.
  • Une interface physique ou logique pour configurer le lien trunk.

Étapes pour configurer un lien trunk

Exemple avec des switches Cisco (IOS)

  1. Configurer les VLANs sur chaque switch : 
    Switch> enable
Switch# configure terminal
Switch(config)# vlan 10
Switch(config-vlan)# name VLAN10
Switch(config-vlan)# exit
Switch(config)# vlan 20
Switch(config-vlan)# name VLAN20
Switch(config-vlan)# exit
  2. Attribuer les ports aux VLANs (optionnel) : Si nécessaire, configurez des ports d'accès pour chaque VLAN : 
    Switch(config)# interface FastEthernet0/1
Switch(config-if)# switchport mode access
Switch(config-if)# switchport access vlan 10
Switch(config-if)# exit

Switch(config)# interface FastEthernet0/2
Switch(config-if)# switchport mode access
Switch(config-if)# switchport access vlan 20
Switch(config-if)# exit
  3. Configurer un lien trunk : Sélectionnez les ports reliant les deux switches et configurez-les en mode trunk : 
    Switch(config)# interface FastEthernet0/24
Switch(config-if)# switchport mode trunk
Switch(config-if)# switchport trunk encapsulation dot1q
Switch(config-if)# exit
  4. Vérifier la configuration :
    • Pour vérifier que le port est en mode trunk : 
      Switch# show interfaces FastEthernet0/24 switchport
    • Pour vérifier les VLANs autorisés sur le trunk : 
      Switch# show running-config

Commandes supplémentaires utiles

  • Restreindre les VLANs autorisés sur le trunk : 
    Switch(config-if)# switchport trunk allowed vlan 10,20
  • Configurer un VLAN natif (non tagué) : 
    Switch(config-if)# switchport trunk native vlan 1
  • Dépannage :
    • Vérifiez l’état des VLANs sur le switch : 
      Switch# show vlan brief
    • Vérifiez l’état des trunks : 
      Switch# show interfaces trunk

Conclusion

Le trunking est une fonctionnalité essentielle pour transporter plusieurs VLANs entre les switches, permettant une segmentation réseau logique tout en optimisant l’utilisation des liens physiques. En suivant les étapes ci-dessus, vous pouvez configurer efficacement un lien trunk entre vos switches et assurer une connectivité VLAN robuste et sécurisée.

N'hésitez pas à adapter les commandes en fonction de votre équipement réseau ou à consulter la documentation de votre fabricant pour des ajustements spécifiques.
