Tutoriel : Protocole OSPF

Introduction

OSPF (Open Shortest Path First) est un protocole de routage à état de liens largement utilisé dans les réseaux internes pour déterminer les chemins les plus courts entre les routeurs. Il fait partie de la famille des protocoles IGP (Interior Gateway Protocol) et fonctionne sur le protocole IP.

Caractéristiques principales

  • Utilise l'algorithme de Dijkstra pour calculer les chemins les plus courts.
  • Prise en charge de la hiérarchie en divisant le réseau en zones (area).
  • Communication efficace grâce à l'utilisation de messages LSA (Link State Advertisements).
  • Convergence rapide et adaptabilité en cas de modification du réseau.

Fonctionnement

  1. Découverte des voisins : Les routeurs identifient les routeurs voisins en échangeant des messages Hello.
  2. État des liens : Chaque routeur crée une base de données d'état de liens (LSDB) qui contient des informations sur les connexions des autres routeurs.
  3. Calcul des chemins : L'algorithme de Dijkstra est utilisé pour générer une table de routage basée sur la LSDB.
  4. Mise à jour : Les LSDB sont synchronisées entre routeurs via des LSA pour refléter les changements dans le réseau.

Structure du réseau OSPF

OSPF organise les réseaux en zones. Chaque zone est connectée à une zone centrale appelée Area 0 ou backbone :

  • Area 0 : Zone principale où transitent les informations entre les autres zones.
  • Autres zones : Zones périphériques connectées à l'Area 0.

Commandes OSPF de base

Voici quelques commandes courantes utilisées pour configurer OSPF sur des routeurs Cisco :

router ospf 1 network 192.168.1.0 0.0.0.255 area 0 network 10.0.0.0 0.0.0.255 area 1 show ip ospf show ip ospf neighbor

Exemple de configuration

# Activer OSPF avec un ID de processus router ospf 1 # Ajouter des réseaux à OSPF network 192.168.10.0 0.0.0.255 area 0 network 10.0.1.0 0.0.0.255 area 1 # Vérifier la configuration show running-config | section ospf # Vérifier les voisins OSPF show ip ospf neighbor

Avantages et inconvénients

Avantages

  • Convergence rapide.
  • Évolutif grâce à la hiérarchie par zones.
  • Prise en charge de réseaux hétérogènes.

Inconvénients

  • Configuration plus complexe que d'autres protocoles comme RIP.
  • Consomme plus de ressources en termes de CPU et de mémoire.
