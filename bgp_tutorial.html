Tutoriel : Protocole BGP

Introduction

BGP (Border Gateway Protocol) est un protocole utilisé pour échanger des informations de routage entre différents systèmes autonomes (AS). Il est essentiel pour le fonctionnement d'Internet en tant que réseau global.

Caractéristiques principales

  • Permet l'interconnexion de multiples réseaux autonomes.
  • Utilise des mises à jour incrémentielles pour réduire la consommation de bande passante.
  • Prise en charge des politiques de routage personnalisées.
  • Fonctionne sur le protocole TCP (port 179).

Fonctionnement

  1. Établissement des sessions : BGP utilise TCP pour établir des sessions entre routeurs (peers).
  2. Échange des routes : Les routeurs échangent des informations sur les préfixes IP qu'ils peuvent atteindre.
  3. Application des politiques : Les politiques définies par les administrateurs influencent les décisions de routage.
  4. Mise à jour : Les changements sont communiqués uniquement lorsqu'ils se produisent.

Commandes BGP de base

Voici quelques commandes courantes pour configurer BGP sur des routeurs Cisco :

router bgp 65001 neighbor 192.168.1.1 remote-as 65002 network 192.168.10.0 mask 255.255.255.0 show ip bgp show ip bgp summary

Exemple de configuration

# Activer BGP avec un numéro d'AS router bgp 65001 # Configurer un voisin BGP neighbor 192.168.1.1 remote-as 65002 # Annoncer un réseau spécifique network 192.168.10.0 mask 255.255.255.0 # Vérifier les routes BGP show ip bgp # Résumé de la session BGP show ip bgp summary

Avantages et inconvénients

Avantages

  • Essentiel pour la connectivité Internet mondiale.
  • Permet une grande flexibilité dans les politiques de routage.
  • Très évolutif pour les grands réseaux.

Inconvénients

  • Complexité de configuration et de maintenance.
  • Sensibilité aux erreurs de configuration, pouvant causer des problèmes à grande échelle.
