QoS (Quality of Service)

QoS (Quality of Service) : Mise en place de politiques pour gérer et prioriser les flux de données critiques dans les réseaux.

Avantages de QoS

  • Priorisation : Assure que les applications critiques reçoivent la bande passante nécessaire.
  • Réduction de la latence : Particulièrement utile pour les applications en temps réel comme la voix et la vidéo.
  • Meilleure utilisation des ressources : Optimise l’allocation de la bande passante sur le réseau.

Configuration de QoS


class-map match-any VoIP
  match protocol rtp

policy-map Prioritize-VoIP
  class VoIP
    priority percent 30

interface GigabitEthernet1/0/1
  service-policy output Prioritize-VoIP

Vérification

Pour vérifier le fonctionnement de la QoS :

  • Cisco : Utilisez la commande show policy-map interface.
  • Linux : Utilisez la commande tc -s qdisc pour afficher les statistiques de la QoS.

Conclusion

QoS garantit que les applications critiques reçoivent les ressources nécessaires pour fonctionner de manière optimale. En appliquant des politiques adaptées, vous pouvez améliorer considérablement les performances globales de votre réseau.
